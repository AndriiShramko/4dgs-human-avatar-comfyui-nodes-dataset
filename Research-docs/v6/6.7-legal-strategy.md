# Section 7: Legal Strategy and Clean-Room Design

## Executive Summary

The v5 legal analysis is **directionally correct** but contains a critical gap: it treats the SMPL licensing problem as having no practical solution other than expensive Meshcapade licensing or abandoning parametric body models entirely. In reality, **two commercially viable, Apache 2.0 licensed alternatives emerged in 2025**: Anny (NAVER Labs) and MHR (Meta). Both are drop-in replacements for SMPL in feed-forward avatar generation pipelines. The gsplat recommendation is confirmed as sound. Clean-Room Design methodology is validated but v5 fails to estimate the **real timeline cost** (+3-6 months to development) and required **organizational infrastructure** (3-team isolation protocol).

## SOTA Analysis 2025-2026

### 7.1 Anny: NAVER Labs Parametric Body Model (Apache 2.0)

Released in 2025, Anny is a comprehensive open-source alternative to SMPL:

**License**: Apache License 2.0 — unrestricted commercial use, modification, redistribution.

**Technical capabilities**:
- Covers **full human lifespan**: infants to elderly in a single unified model
- Interpretable controls: age, height, weight, muscle sliders
- Based on MakeHuman assets and WHO anthropometric data (privacy-friendly — no real body scans)
- Competitive or superior results on human mesh recovery benchmarks
- v0.3 (February 2026) adds **SMPL-X topology compatibility** for interoperability

**GitHub**: https://github.com/naver/anny (active development, tutorials, interactive demo)

**Key advantage for our project**: SMPL-X topology compatibility means existing SMPL-based pose estimation models can work with Anny's body representation without retraining.

### 7.2 MHR: Meta Momentum Human Rig (Apache 2.0)

Released in 2025, MHR is Meta's parametric 3D human body model:

**License**: Apache License 2.0 — full commercial freedom.

**Technical capabilities**:
- 45 shape parameters (body identity)
- 204 pose parameters (articulation)
- 72 facial expression parameters
- 7 levels of detail (LOD 0-6) for performance flexibility
- GPU-accelerated PyTorch implementation for real-time applications

**GitHub**: https://github.com/facebookresearch/MHR

**Key advantage for our project**: LOD system allows using lightweight representation during training (fast iteration) and high-detail at inference (quality output). Facial expressions are included — SMPL requires separate SMPL-X for face/hands.

### 7.3 Comparison: SMPL vs Anny vs MHR

| Feature | SMPL/SMPL-X (MPI) | Anny (NAVER) | MHR (Meta) |
|---------|-------------------|--------------|------------|
| **License** | Non-commercial (academic only) | **Apache 2.0** | **Apache 2.0** |
| **Commercial cost** | ~150K EUR/yr (Meshcapade) | **Free** | **Free** |
| **Shape params** | 10 (SMPL) / 10 (SMPL-X) | Interpretable (age, weight, height) | 45 |
| **Pose params** | 72 (SMPL) / 165 (SMPL-X) | Compatible with SMPL-X | 204 |
| **Face/hands** | SMPL-X only | Via SMPL-X topology adapter | 72 expression params built-in |
| **LOD support** | No | No | **7 LOD levels** |
| **GPU acceleration** | Community implementations | PyTorch | **Native PyTorch** |
| **Training data** | Real 3D body scans | MakeHuman + WHO data | Meta internal data |
| **Ecosystem maturity** | Very high (10+ years) | Growing (1+ year) | Growing (1+ year) |
| **Patent risk** | US10395411B2 | **None** (Apache patent grant) | **None** (Apache patent grant) |

**Recommendation**: Use **MHR** as primary body model (richer parameterization, LOD, facial expressions), with **Anny** as backup/validation alternative.

### 7.4 gsplat Legal Analysis (Confirmed)

gsplat (Nerfstudio project) under Apache License 2.0:

- **Section 2 (Grant of Copyright License)**: Perpetual, worldwide, irrevocable right to reproduce, distribute, and create derivative works
- **Section 3 (Grant of Patent License)**: Contributors grant perpetual patent license for any patents embodied in their contributions
- **Section 4 (Redistribution)**: Must retain copyright notices and NOTICE file; no obligation to open-source derivative works
- **No copyleft/viral requirements**: Unlike GPL, does not require opening proprietary code

**Technical validation**: gsplat provides packed sparse tensor mode (`packed=True`) critical for handling variable-length Gaussian sets, sparse gradient support, N-dimensional feature rendering, and distributed multi-GPU rasterization.

### 7.5 Clean-Room Design: Realistic Assessment

**What Clean-Room protects against**:
- Copyright infringement claims (code copying)
- Derivative work claims (code modification)
- Trade secret misappropriation

**What Clean-Room does NOT protect against**:
- Patent infringement (independent creation is not a defense)
- Breach of Terms of Service (if team members signed academic licenses before joining)
- Inadvertent contamination (developer saw code on GitHub before the clean-room process started)

**Realistic timeline impact**:

| Clean-Room Activity | Duration | Notes |
|-------------------|----------|-------|
| Specification Team: analyze papers → write math specs | 4-6 weeks | Per major module (attention, rasterizer, loss functions) |
| Legal review of specifications | 1-2 weeks | Per spec document |
| Implementation Team: code from specs | 6-10 weeks | Per major module |
| Verification and testing | 2-4 weeks | Functional equivalence testing |
| **Total per module** | **13-22 weeks** | |
| **Full pipeline (parallel modules)** | **4-6 months** | With 3-4 parallel module teams |

**Organizational requirements**:
- Physically separate development environments (separate Git repos, separate Slack channels)
- No cross-team code review between Specification and Implementation teams
- Legal audit trail: timestamped commits, specification sign-off documents
- Background checks: ensure Implementation Team members haven't previously worked with restricted code

## Critical Analysis

### Risk 1: Anny/MHR Ecosystem Maturity

SMPL has been the industry standard for 10+ years. Anny and MHR are 1-2 years old. Risks include:
- Fewer pre-trained pose estimation models compatible with Anny/MHR topology
- Less community support and debugging resources
- Potential breaking changes in early versions

**Mitigation**: Anny v0.3 explicitly adds SMPL-X topology compatibility, allowing use of existing SMPL-X pose estimators. MHR's PyTorch implementation follows standard skinning conventions.

### Risk 2: Patent Landscape Beyond SMPL

Even after removing SMPL, the 4DGS avatar pipeline may encounter other patents:
- Differentiable rendering patents (various holders)
- Attention mechanism patents in transformer architectures
- Body pose estimation method patents

**Mitigation**: Conduct a focused freedom-to-operate (FTO) search on core algorithmic components before MVP launch. Budget: ~$15,000-25,000 for a specialized IP attorney.

### Risk 3: Clean-Room Process Failure

Common failure modes:
- Developer on Implementation Team googles a function name and finds the restricted repository
- Specification Team includes too-specific implementation details that mirror original code
- Time pressure leads to shortcuts (skipping legal review)

**Mitigation**: Strict browser monitoring on Implementation Team machines (block specific GitHub repos). Automated code similarity scanning (using tools like MOSS or JPlag) on every commit.

### Risk 4: Meta's MHR Future Licensing

Apache 2.0 is irrevocable — Meta cannot retroactively change the license for already-released code. However:
- Future versions of MHR could use a different license
- Meta could stop maintaining the project
- Patent landscape around Meta's body model research could evolve

**Mitigation**: Fork MHR at a specific version and maintain internally. Apache 2.0's irrevocable grant protects the forked version permanently.

### Clean-Room Implementation Plan

**Module 1: Voxel-Aligned Transformer** (8-12 weeks)
- Spec from: VolSplat paper, 4DGT paper (math only)
- Implementation: Custom PyTorch nn.Module, no reference code

**Module 2: DCT Trajectory Module** (4-6 weeks)
- Spec from: VeGaS paper, DCT math textbooks
- Implementation: Pure tensor operations, well-understood math

**Module 3: GNN Dynamics Module** (6-8 weeks)
- Spec from: CloDS paper, GNN theory
- Implementation: PyTorch Geometric or custom message passing

**Module 4: Composite Loss Function** (2-3 weeks)
- Spec from: SHARP paper, LPIPS paper, DISTS paper
- Implementation: Mostly using existing Apache/MIT-licensed loss libraries (lpips package is BSD)

**Module 5: Compression Pipeline** (4-6 weeks)
- Spec from: MEGA paper, P-4DGS paper
- Implementation: Custom codec, no reference code from MEGA repo

**Module 6: ComfyUI Integration** (4-6 weeks)
- Spec from: ComfyUI API documentation (GPL, but node API is public)
- Implementation: Custom node package, isolated from ComfyUI internals

## Open Questions

1. **Anny vs MHR head-to-head**: Which produces better results for our specific use case (studio rig, diverse subjects, 4K quality)? Requires A/B testing.
2. **Pose estimator compatibility**: Which open-source pose estimator works best with Anny/MHR topology? Candidates: ViTPose (Apache 2.0), 4DHumans (license TBD).
3. **Diffusion teacher licensing**: If we use Stable Diffusion XL as the diffusion teacher during training, do the generated intermediate latents "contaminate" our model weights? Legal opinion needed.
4. **ComfyUI GPL boundary**: ComfyUI itself is GPL-3.0. Our custom nodes are separate packages loaded at runtime. Does this constitute a "derivative work"? Standard legal opinion says no (plugins are not derivatives), but should be confirmed.
5. **International patent landscape**: The FTO search should cover US, EU, and Asian patent offices. SMPL patent (US10395411B2) may have equivalents in other jurisdictions.
