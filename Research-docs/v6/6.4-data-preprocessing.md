# Section 4: Petabyte Data Preprocessing Optimization

## Executive Summary

The v5 plan's reliance on Fisher Information Matrix (FIM) diagonal approximation for active view selection is **conceptually sound but mathematically questionable** for heterogeneous 4DGS parameters. Diagonal FIM ignores cross-parameter correlations (e.g., between quaternion rotation and SH coefficients) that are critical in the 161-dimensional Gaussian parameter space. More practical alternatives exist: the "Squisher" method (2025) obtains diagonal Fisher **for free** from Adam's gradient accumulators, and ActiveInitSplat (2025) uses simpler density/occupancy criteria that may be more robust. The recommended approach is a **multi-criteria filtering pipeline** combining lightweight gradient-based informativeness with geometric coverage heuristics.

## SOTA Analysis 2025-2026

### 4.1 The Mathematical Problem with Diagonal FIM for 4DGS

The Fisher Information Matrix for a model with parameters θ is defined as:

```
F(θ) = E[∇_θ log p(x|θ) · ∇_θ log p(x|θ)^T]
```

For a 4DGS scene with N Gaussians and D=161 parameters each, the full FIM has dimension (N·D)² — over 4×10¹⁸ elements for 13M Gaussians. This is obviously intractable.

**Diagonal approximation** keeps only the diagonal elements, assuming parameter independence:

```
F_diag(θ) = diag(E[(∇_θ log p(x|θ))²])
```

This assumption is **violated** in 4DGS because:

1. **Rotation-Scale coupling**: Quaternion rotation (q) and scale (s) jointly determine the covariance matrix Σ = RSS^TR^T. Changing rotation affects how scale manifests geometrically. The off-diagonal F(q,s) blocks carry essential information.

2. **Position-SH coupling**: Moving a Gaussian (changing μ) changes which rays intersect it, altering the effective SH evaluation direction. Position and color are not independent.

3. **Opacity-everything coupling**: A Gaussian's opacity determines its gradient weight for ALL other parameters. Zero-opacity Gaussians should contribute zero information, but diagonal FIM doesn't capture this gating effect.

**Research evidence**: Li et al. (2024) "Trade-Offs of Diagonal Fisher Information Matrix Estimators" demonstrates that diagonal estimators have "varying accuracy depending on parameter group non-linearity" and recommends considering group structure.

### 4.2 Squisher: Free Fisher from Adam (ICML 2025)

The Squisher method eliminates the need for explicit FIM computation entirely:

**Key insight**: Adam optimizer maintains a running estimate of E[g²] (squared gradient exponential moving average) for each parameter. This is mathematically equivalent to the diagonal of the empirical Fisher Information Matrix.

```
v_t = β₂ · v_{t-1} + (1-β₂) · g_t² ≈ F_diag(θ)
```

**Advantages**:
- Zero additional computation — reuses values already computed by Adam
- Updated at every training step — always reflects current model state
- Validated across 5 applications with comparable performance to explicit Fisher estimation

**Application to our pipeline**: During training on the 90-camera rig, extract Adam's v_t for each viewpoint's contribution. Viewpoints with consistently low v_t (low gradient magnitude → low information content) are candidates for filtering.

### 4.3 ActiveInitSplat: Simpler Criteria (2025)

ActiveInitSplat proposes active image selection for Gaussian Splatting initialization without any Fisher machinery:

**Criteria**:
1. **Density criterion**: Select views that maximize spatial coverage of the Gaussian point cloud
2. **Occupancy criterion**: Prefer views that observe under-represented regions of the scene

**Results**: Significant performance improvement over passive dense-image baselines, demonstrating that geometric coverage heuristics can be as effective as information-theoretic approaches for view selection.

### 4.4 POp-GS: Block-Diagonal Covariance (2025)

POp-GS improves upon diagonal FIM by using **block-diagonal covariance approximation**:

- Groups parameters by Gaussian (each Gaussian's 161 parameters form one block)
- Captures within-Gaussian correlations (rotation-scale, position-SH coupling)
- Ignores between-Gaussian correlations (reasonable assumption for spatially distant Gaussians)

Optimality criteria: P-Optimality, T-Optimality, D-Optimality for experimental design.

**Trade-off**: More accurate uncertainty estimation, but higher computational cost per evaluation.

### 4.5 Active Learning with Semantic Features (2025)

Recent work uses **semantic feature variance** of Gaussian attributes to identify informative regions:

- Regions where Gaussian attributes have high variance across nearby viewpoints are "uncertain"
- Incorporates robust pose optimization to handle practical noise
- More interpretable than FIM-based approaches

## Critical Analysis

### Risk 1: Over-engineering the Selection Pipeline

The v5 plan treats active view selection as a complex optimization problem requiring advanced information theory. In practice, for a fixed 90-camera rig with known geometry:

- **Many cameras see nearly identical content** (adjacent cameras in a dense rig)
- **Simple geometric heuristics** (angular diversity, depth coverage) may suffice
- The computational cost of sophisticated FIM analysis may exceed the savings from filtering

A pragmatic approach: start with simple geometric sampling, add information-theoretic refinement only if needed.

### Risk 2: Temporal Redundancy vs Spatial Redundancy

The v5 analysis focuses on spatial redundancy (too many cameras). But temporal redundancy may be a larger source of waste:
- 60 FPS × 10 seconds = 600 frames per camera
- Consecutive frames of a slowly moving subject are nearly identical
- **Temporal subsampling** (e.g., adaptive frame rate based on motion magnitude) could provide larger savings than view selection

### Risk 3: Filtering Too Aggressively

The 70% filtering target is aggressive. Risks include:
- Missing rare viewpoint-specific details (e.g., a logo on the back visible from only 2 cameras)
- Insufficient coverage of fast-moving regions where more temporal samples are needed
- Bias toward "easy" regions that have low uncertainty simply because they're featureless

**Mitigation**: Progressive filtering — start with conservative 30% reduction, evaluate quality, increase gradually.

## Architectural Recommendations for MVP

### Multi-Stage Filtering Pipeline

```
Stage 1: Geometric Diversity Filter (fast, rule-based)
  - Select cameras maximizing angular coverage (minimum angular distance between selected cameras)
  - For 90 cameras → select ~30 cameras evenly distributed around the subject
  - Cost: negligible (precomputed camera geometry)
  
Stage 2: Temporal Subsampling (motion-adaptive)
  - Compute optical flow magnitude between consecutive frames
  - High-motion frames: keep at full 60 FPS
  - Low-motion frames: subsample to 15 FPS
  - Expected reduction: ~40% of total frames
  
Stage 3: Gradient-Based Informativeness (Squisher, training-time)
  - During first N training epochs, accumulate Adam's v_t per viewpoint
  - After warmup, drop viewpoints with consistently low gradient magnitude
  - Additional reduction: ~20% of remaining viewpoints
  
Total expected reduction: ~65-75%
```

### Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Primary filtering method | Geometric diversity + temporal subsampling | Simple, interpretable, no compute overhead |
| Informativeness metric | Adam v_t (Squisher) | Free, always up-to-date, proven equivalent to diagonal FIM |
| FIM computation | Not used explicitly | Diagonal approximation is questionable for heterogeneous 4DGS parameters; Squisher provides same signal for free |
| Block-diagonal FIM (POp-GS) | Deferred to v2 | Higher cost, unclear benefit over Squisher for our fixed-rig setup |
| Filtering schedule | Progressive (start conservative) | Avoids irreversible information loss |
| Validation | Monitor PSNR on held-out viewpoints after filtering | Detect quality degradation early |

## Open Questions

1. **Optimal camera count**: What is the minimum number of cameras (from 90) needed for acceptable quality? Requires ablation study (90 → 60 → 30 → 15 → 8).
2. **Motion-adaptive temporal sampling**: Which motion metric — optical flow, pose change, or reconstruction error — best predicts frame informativeness?
3. **Cross-subject transfer**: Can the informativeness ranking learned from one subject generalize to other subjects, or must it be recomputed?
4. **Budget allocation**: Should we allocate more cameras to difficult regions (face, hands) and fewer to simple regions (back), or use uniform sampling?
5. **Online vs offline**: Should filtering be a one-time preprocessing step or adapt during training as the model improves?
